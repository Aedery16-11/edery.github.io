Présentation du projet

Au cours de mon stage, j'ai travaillé sur un projet visant à créer une application de gestion des habilitations des employés, dont je détaillerai le fonctionnement.

Mes Missions Pendant le Stage

  • Développement d'une application web avec Flask.
  • Gestion des données avec Google BigQuery.
  • Utilisation de bibliothèques Python comme pandas et pandasql.
  • Configuration de l'environnement de développement avec un environnement virtuel Python.
  • Création de templates HTML pour l'affichage des données.

Technologies principales utilisées

Ce projet a été principalement réalisé avec Flask, qui est un micro-framework permettant de créer des applications web simples.

Les données étaient conservées dans une base de données Google BigQuery, un service de Google Cloud Platform (GCP) conçu pour exécuter des requêtes SQL sur de vastes ensembles de données.

L'utilisation de librairies Python telles que pandas et pandasql m'a permis de filtrer et de formater les données avant de les afficher. "pandas" est une bibliothèque très utilisée pour la manipulation et l'analyse de données Excel, offrant un type de structure de données performant : le DataFrame. "pandasql", quant à lui, permet d'exécuter des requêtes SQL directement sur des DataFrames pandas, facilitant ainsi le traitement des données au sein de l'application. En plus de ces technologies, j'ai dû configurer l'environnement de développement en utilisant un environnement virtuel Python. Cela permet de gérer les dépendances/modules spécifiques au projet de manière isolée. La gestion des dépendances était effectuée à l'aide d'un fichier requirements.txt, qui liste toutes les librairies, frameworks nécessaires pour faire fonctionner l'application, et ce, dans la version dans laquelle elles ont été installées pour développer le projet.

Fonctionnement du projet

Cette application, construite en Flask, se connecte à une base de données Google BigQuery ainsi qu'à un fichier excel et .json, le tout dans le but d'en tirer des informations qu'elle affichera sur la page web à travers des templates HTML. L'utilisateur peut choisir un secteur (Marketing, Data..), un groupe de ressources, c'est-à-dire les ressources auxquelles le collaborateur aura accès (Google Cloud Platform [GCP], SAS [Statistical Analysis System]...) et enfin le nom de l'employé en question.

Cependant, étant donné qu'il ne s'agit pas de l'équipe informatique qui dispose du droit de changer l'habilitation de quelqu'un, l'application ne la change pas elle-même, mais va plutôt rassembler les informations entrées, les manipuler afin de trouver certaines données dans le fichier excel à l'aide de certains outils, tel que "pandas", et inclure ces données dans un mail pré-écrit qui sera automatiquement envoyé à l'équipe disposée à modifier les habilitations.

Compétences développées

Grâce à ce stage, j'ai pu améliorer ma maîtrise de Flask, mais aussi de la gestion des requêtes et des templates. En plus des compétences techniques, j'ai acquis une vision plus globale du déroulement d'un projet et de ma contribution à celui-ci.

Je prends désormais conscience qu'un gros travail de recherche doit être fourni. Cela inclut non seulement la recherche d'outils adaptés au projet, mais aussi l'apprentissage rapide de leur utilisation. J'ai donc appris à rechercher correctement sur internet, à mieux comprendre les documentations, ainsi que de m'aider de diverses ressources, telles que des vidéos explicatives ou des tutoriels en ligne, et ce, dans le but d'adapter ce que je vois à ma situation.
